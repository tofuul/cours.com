NSI – Chapitre 1

Codage et représentation des données

💾 Principe fondamental

Dans un ordinateur, toutes les données sont stockées sous forme de bits (0 et 1).

Les nombres, les textes, les images ou les sons doivent donc être codés en binaire.

La mémoire étant finie, tous les nombres ne sont pas représentables.

🔢 Écriture dans une base quelconque

Un nombre écrit en base b est une somme de puissances de b.

Exemple en base 10 : 2985 = 2×10³ + 9×10² + 8×10 + 5

Exemple en base 2 ou 3 : même principe, mais avec d’autres puissances.

En base 2, seuls les chiffres 0 et 1 sont utilisés.

🔁 Conversion décimal → binaire

Pour convertir un entier en binaire, on effectue des divisions successives par 2.

Les restes, lus de bas en haut, forment l’écriture binaire.

Le bit le plus à gauche est le bit de poids fort, celui de droite le bit de poids faible.

8 bits forment un octet.

🧠 Représentation en machine

Avec n bits, on peut coder les entiers naturels de :

0 à 2ⁿ − 1

  • 8 bits → de 0 à 255
  • 16 bits → de 0 à 65 535
  • 32 bits → jusqu’à ≈ 4,3 milliards

Un calcul peut dépasser la capacité maximale : on parle de dépassement de capacité (overflow).

➕ Addition binaire

L’addition binaire fonctionne comme en base 10, avec des retenues.

Règle clé : 1 + 1 = 10 (0 avec retenue 1).

Les ordinateurs utilisent principalement l’addition pour effectuer tous les calculs.

➖ Codage des entiers relatifs

Un entier relatif peut être codé de plusieurs façons :

  • Bit de signe : 1 bit indique le signe
  • Complément à 2 : méthode standard en machine
  • Excédent : décalage d’une valeur fixe

Le complément à 2 permet de simplifier les opérations arithmétiques.

📐 Codage des nombres réels

Un nombre réel ne peut pas toujours être représenté exactement.

On utilise une approximation binaire.

Les réels sont codés en virgule flottante, sur le modèle scientifique :

x = signe × mantisse × 2exposant

📏 Norme IEEE 754

En Python (type float), les réels suivent la norme IEEE 754.

  • 1 bit de signe
  • 11 bits d’exposant
  • 52 bits de mantisse

Certains nombres comme 0.1 ne sont pas codables exactement → erreurs d’arrondi.

⚠️ Erreurs numériques en Python

Des calculs simples peuvent donner des résultats inattendus :

(0.2 + 0.1) − 0.3 ≠ 0

Ce phénomène est dû aux limites du codage binaire des réels.

🔤 Codage des caractères

Un texte est une suite de caractères, chacun codé par un nombre.

  • ASCII : 128 caractères
  • UTF-8 : permet tous les alphabets

UTF-8 est compatible avec ASCII et utilise parfois plusieurs octets.
