NSI – Chapitre 0

Les bases de la programmation

🧠 Variables et affectation

Une variable permet de stocker une valeur. L’affectation consiste à donner une valeur à une variable à l’aide du symbole ← (ou = en Python).

Le contenu affecté dépend de la présence des guillemets : avec guillemets, on affecte une chaîne de caractères ; sans guillemets, on affecte le contenu d’une variable.

L’ordre des instructions est essentiel : la dernière affectation écrase les précédentes.

🧮 Expressions et opérateurs

Une expression est un ensemble de valeurs reliées par des opérateurs et qui donne une seule valeur.

À gauche d’une affectation : uniquement un nom de variable. À droite : une expression du même type.

  • Opérateurs numériques : +, −, *, /
  • Python : ** (puissance), // (division entière), % (modulo)
  • Les parenthèses permettent de modifier les priorités de calcul

🔤 Chaînes de caractères et concaténation

Une chaîne de caractères est un texte entre guillemets.

La concaténation permet d’assembler plusieurs chaînes.

En Python, on utilise l’opérateur + pour concaténer.

Le TP montre qu’on peut créer plusieurs phrases différentes en changeant l’ordre des chaînes.

⌨️ Lecture et écriture

input() permet de lire une valeur entrée par l’utilisateur.

Par défaut, input renvoie une chaîne de caractères : il faut préciser le type (int, float).

print() permet d’afficher un résultat à l’écran.

Il est recommandé d’afficher un message avant une saisie pour guider l’utilisateur.

🔀 Tests et conditions

Un test permet d’exécuter des instructions seulement si une condition est vraie.

  • if : test simple
  • if / else : alternative
  • if / elif / else : plusieurs cas possibles

Une condition est une comparaison entre deux valeurs.

⚖️ Opérateurs de comparaison et logiques

  • Comparaison : ==, !=, <, >, <=, >=
  • Logiques : and, or, not

Le and exige que toutes les conditions soient vraies.

Le or est vrai si au moins une condition est vraie.

Le not inverse une condition.

Ces opérateurs sont utilisés dans les exercices (imposition, produit positif/négatif, dates).

🔁 Boucles

Les boucles permettent de répéter des instructions.

Boucle while : on ne connaît pas à l’avance le nombre de répétitions.

Boucle for : on connaît le nombre de répétitions à l’avance.

Attention aux boucles infinies : le test doit devenir faux à un moment.

⚠️ Erreurs classiques à éviter

  • Modifier la variable compteur dans une boucle for
  • Oublier de faire évoluer une condition dans un while
  • Mélanger types numériques et chaînes sans conversion

📌 Ce que montrent les corrections du TP

  • Manipulation de chaînes et concaténation
  • Utilisation de conditions simples et composées
  • Calculs numériques automatisés
  • Parcours de valeurs avec for et while

✅ À retenir absolument

  • Une variable stocke une valeur
  • Une affectation dépend de l’ordre des instructions
  • Une condition compare deux valeurs
  • Les boucles servent à répéter des actions
  • La rigueur est essentielle en programmation
