NSI – Chapitre 3

Architecture des ordinateurs

🧠 Idée centrale du chapitre

Un ordinateur est une machine qui traite de l’information en exécutant des instructions sur des données, uniquement à partir de deux états : 0 et 1.

Tout le fonctionnement repose sur des composants électroniques organisés de manière logique.

⚡ Pourquoi le binaire ?

À la base d’un ordinateur se trouve le transistor.

Il se comporte comme un interrupteur :

  • courant qui passe → état haut → 1
  • courant qui ne passe pas → état bas → 0

Un ordinateur fonctionne donc uniquement en base 2 (binaire).

🧮 Algèbre de Boole et logique

Les traitements réalisés par l’ordinateur reposent sur des opérations logiques issues de l’algèbre de Boole.

Un circuit logique reçoit des entrées (0 ou 1) et produit une sortie (0 ou 1).

On distingue :

  • Circuits combinatoires : sortie dépend uniquement des entrées
  • Circuits séquentiels : sortie dépend aussi du temps et des états précédents

🔌 Portes logiques fondamentales

  • NON (NOT) : inverse la valeur
  • ET (AND) : vrai si les deux entrées sont vraies
  • OU (OR) : vrai si au moins une entrée est vraie
  • OU exclusif (XOR) : vrai si une seule entrée est vraie

En combinant ces portes, on peut créer des circuits plus complexes comme des additionneurs.

➕ Additionneur binaire

Un additionneur permet d’additionner des bits.

Il produit :

  • une somme (S)
  • une retenue (Cout)

Les additionneurs multi-bits sont construits à partir de plusieurs additionneurs 1 bit.

💾 La mémoire vive (RAM)

La RAM stocke les données et les programmes en cours d’exécution.

  • Stockage sous forme de bits (0/1)
  • Organisation en octets (8 bits)
  • Chaque case mémoire possède une adresse

La RAM est une mémoire volatile : tout est perdu quand on éteint l’ordinateur.

🧠 Le processeur (CPU)

Le processeur est le cerveau de l’ordinateur.

Il est composé de :

  • UAL : réalise les opérations arithmétiques et logiques
  • Unité de contrôle : organise l’exécution des instructions
  • Registres : stockage temporaire très rapide

🏗️ Architectures des ordinateurs

Architecture de Von Neumann :

  • Même mémoire pour données et instructions
  • Très répandue dans les ordinateurs actuels

Architecture Harvard :

  • Mémoire données et mémoire programme séparées
  • Plus rapide mais plus complexe

🔁 Cycle d’exécution d’un programme

Le processeur fonctionne en boucle :

  1. Lire l’instruction en mémoire
  2. La décoder
  3. L’exécuter
  4. Passer à la suivante

Ce cycle est synchronisé par une horloge (GHz).

🧾 Langage machine et assembleur

Le processeur ne comprend que le binaire.

Le langage assembleur permet une écriture plus lisible :

  • LDA : charger une valeur
  • ADD : additionner
  • STA : stocker en mémoire
  • BRA / BRZ : sauts

Un programme Python est finalement traduit en instructions machine.

🧪 Ce que montre le TP

  • Fonctionnement des opérateurs booléens (and, or, not)
  • Construction d’un XOR à partir de portes simples
  • Simulation d’un additionneur 1 bit
  • Lien direct entre logique électronique et Python

✅ À retenir absolument

  • Un ordinateur fonctionne uniquement avec 0 et 1
  • Les transistors sont à la base de tout
  • Les portes logiques permettent les calculs
  • La RAM est volatile
  • Le processeur exécute les instructions séquentiellement
