NSI – Chapitre 2

Structures séquentielles

🧠 Idée centrale du chapitre

Quand on manipule beaucoup de données ou des données liées entre elles, on utilise des structures séquentielles : chaînes de caractères, tuples, listes et dictionnaires.

Ces structures permettent de regrouper, parcourir et traiter efficacement des données.

🔤 Chaînes de caractères (str)

Une chaîne de caractères est une suite ordonnée de caractères.

C’est une structure séquentielle non modifiable (non mutable).

  • Accès par indice : chaine[i]
  • Longueur : len(chaine)
  • Tranches (slicing) possibles
  • Concaténation et répétition possibles

Toute tentative de modification directe provoque une erreur TypeError.

📦 Tuples (séquences)

Un tuple est une structure séquentielle définie par des éléments séparés par des virgules.

Comme les chaînes, les tuples sont non modifiables.

Ils sont utiles pour regrouper des valeurs liées entre elles (coordonnées, résultats multiples…).

Les parenthèses sont parfois implicites.

📊 Listes (tableaux)

Une liste est une structure séquentielle modifiable.

Elle peut contenir des données de tout type et permet des traitements par boucles.

  • Accès par indice à partir de 0
  • Parcours avec for ou while
  • Ajout dynamique avec append

Les indices doivent être des entiers compris entre 0 et la taille − 1.

🧩 Listes multidimensionnelles

Une liste peut contenir d’autres listes → tableaux à plusieurs dimensions.

On y accède avec plusieurs indices : tab[i][j].

Ce type de structure est utilisé pour modéliser des grilles, des plateaux de jeu, des matrices…

Le choix du nombre de dimensions dépend du problème à résoudre.

🔁 Opérations communes aux structures séquentielles

  • Accès par indice
  • Tranches (slicing)
  • len, min, max
  • Appartenance : x in s
  • Concaténation et répétition

Ces opérations fonctionnent sur les chaînes, les listes et les tuples.

✂️ Slicing (saucissonnage)

Le slicing permet d’extraire une partie d’une structure séquentielle :

liste[début : fin : pas]

  • Les bornes peuvent être omises
  • Un pas négatif permet d’inverser l’ordre
  • liste[:] permet de copier une liste

C’est un outil très puissant pour manipuler les données.

🧬 Copie de listes

Une affectation simple crée un alias (même adresse mémoire).

Modifier une liste modifie alors toutes ses références.

Pour éviter cela :

  • liste[:] → copie superficielle
  • deepcopy → copie profonde (si listes imbriquées)

⚡ Compréhension de liste

La compréhension de liste permet de créer une liste à partir d’une autre :

[expression for x in liste if condition]

C’est une écriture courte, lisible et efficace.

🗂️ Dictionnaires

Un dictionnaire associe une clé à une valeur.

Contrairement aux listes, l’accès ne se fait pas par indice mais par clé.

  • Clés uniques
  • Valeurs de tout type
  • Accès rapide aux données

Ils sont très utilisés pour structurer des informations complexes.

🧪 Ce que montre le TP

  • Parcours de tableaux avec for et while
  • Calculs sur les listes (somme, maximum, position)
  • Création de nouvelles listes à partir d’anciennes
  • Recherche simple et recherche dichotomique
  • Génération de tableaux aléatoires

Le TP montre comment exploiter concrètement les structures séquentielles avec des algorithmes classiques.

✅ À retenir absolument

  • Les structures séquentielles stockent des ensembles de données
  • str et tuple → non modifiables
  • list → modifiable
  • Le slicing est un outil clé
  • Les dictionnaires utilisent des clés
